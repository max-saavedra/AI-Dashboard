## 📌 What Was Fixed

### The Problem
POST /api/v1/summary endpoint was returning 500 Internal Server Error:
    greenlet_spawn has not been called; can't call await_only() here

### Root Cause
SQLAlchemy 2.0 async strict mode + lazy loading after async context change:
    1. Load ORM object (Dashboard) 
    2. Call AI (long async operation ~2-5 seconds)
    3. After AI returns, access ORM JSONB fields
    4. SQLAlchemy tries lazy-loading in wrong greenlet context
    5. Crash: greenlet_spawn error → 500 response

### The Solution (2 changes to summary.py)

CHANGE #1 - Lines 55-57: Copy data before async operations
    ❌ OLD: kpi_payload: dict = dashboard.kpi_data or {}
    ✅ NEW: kpi_payload: dict = dict(dashboard.kpi_data or {})

CHANGE #2 - Line 99: Use commit() instead of flush()
    ❌ OLD: await db.flush()
    ✅ NEW: await db.commit()

### Impact
✅ /summary endpoint returns 200 OK (not 500)
✅ Summary text generated correctly
✅ No greenlet_spawn errors
✅ Same response time (2-5 seconds)
✅ Token optimization unaffected (-62% still active)
